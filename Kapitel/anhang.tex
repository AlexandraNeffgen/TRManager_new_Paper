\subsection{User-Stories}
User-Stories Grundfunktionen
\begin{itemize}
  \item Import und Export der Daten / Datensicherung (Priorität 1)
  \begin{itemize}
    \item Zu Beginn jedes Schuljahres werden die Schülerdaten aus der externen
    SchILD- Datenbank (Verwaltungsnetzwerk) bezogen und im CSV-Format in das
    Programm (Schülernetzwerk) geladen.
	\item Vorher sollen alle Daten, die das Programm im letzten Schuljahr erfasst
			hat, zunächst exportiert und dann im Programm selbst gelöscht werden.
	\item Der Datentransfer (Import/Export) erfolgt über die USB-Schnittstelle.
	\item Ein Export der vom Programm erfassten Daten soll zu jeder Zeit möglich
			sein. Dabei sollen bei aktiven Filtern / Sortierungen (s. o.) auch nur die
			ausgewählten Daten exportiert werden können.
	\item Zusätzlich soll eine Datensicherungsfunktion implementiert werden, die es
			ermöglicht, alle Daten direkt mit Datumsangabe auf einen externen Datenträger
			(USB) zu speichern.
	\item Diese Funktionen sollen ausschließlich dem Administrator zur Verfügung
			stehen.
  \end{itemize}
  \item Login Daten (Priorität 1)
  \begin{itemize}
    \item Jeder Lehrer soll sich im Programm mit seinem Kürzel (ohne Passwort)
    anmelden können, um die Informationen des Laufzettels in die grafische
    Oberfläche eintragen zu können.
  \end{itemize}
  \item Besuch eines Schülers eintragen (Priorität 1)
  \begin{itemize}
    \item Sobald ein Schüler den Trainingsraum betritt, wird zunächst zwischen
    neuen Vorfällen und Rückkehrer unterschieden. Dies geschieht unabhängig vom
    Programm durch den betreuenden Lehrer.
    \item Für Rückkehrer siehe Rückkehrer
    \item Bei einem neuen Vorfall wird ein neuer Incident über ein Menü
    eingegeben. Dabei werden folgende Daten gespeichert:
    \begin{itemize}
      \item Name, Klasse des Schülers, schickender Lehrer und eine Bemerkung zu
      dem Vorfall eingetragen.
      \item Bei der Bemerkung soll zusätzlich zu einem Freitextfeld auch eine
      Auswahlliste mit Standardvorgehen bereitgestellt werden.
    \end{itemize}
    \item Ist der Vorfall eingetragen wird eine fortlaufende
    Überweisungsscheinnummer angezeigt, welche vom betreuenden TR-Lehrer auf
    den Überweisungsschein eingetragen wird. Diese wird im Incident ebenfalls
    gespeichert, aber nicht durch den User ausgewählt.
  \end{itemize}
  \item Vervollständigung (Priorität 1)
  \begin{itemize}
    \item Bei Eingabe der Namen sollen Vorschläge zur Vervollständigung
    angezeigt werden. Bei keinem Vorschlag soll die Neuanlage eines Schülers
    erfolgen können.
	\item Zur Erleichterung der korrekten Vervollständigung soll zuerst die
	Klassenbezeichnung des Schülers eingegeben werden können, um somit die
	Vervollständigungsvorschläge zu minimieren und den korrekten Schüler
	vorgeschlagen zu bekommen.
  \end{itemize}
  \item Behandlungen bei drei bzw. fünf Besuchen (Priorität 1)
  \begin{itemize}
    \item Nach Hinzufügen eines Trainingsraumbesuches soll ersichtlich werden,
    wie oft der Schüler in dem jeweiligen Schuljahr bereits im Trainingsraum
    war.
	\item Beim dritten bzw. fünften Besuch soll als Popup ersichtlich werden, dass
	ein Elterngespräch bzw. eine Konferenz notwendig ist.
  \end{itemize}
  \item Behandlung Rückkehrer TR aufgrund von Ablehnung Rückkehrplan (Priorität
  1)
  \begin{itemize}
    \item Kommt ein Schüler ohne neuen Überweisungsschein zurück in den TR, muss
    dies vermerkt werden. Diese Rückkehr ist KEIN neuer TR-Besuch.
    (Rückkehrer-Button)
	\item Gleiche Scheinnummer, neue Incidentnummer
  \end{itemize}
  \item Administrationsrechte (Priorität 1)
  \begin{itemize}
    \item Es soll ein Administrator-Account mit Passwort angelegt werden, der
    als alleinige Instanz die Möglichkeit haben soll, zeitlich unabhängiges
    Hinzufügen und Verändern bereits eingegebener Daten durchzuführen.
  \end{itemize}
  \item Schülerdatenabruf / Information über Besuche im Trainingsraum (TR)
  (Priorität 2)
  \begin{itemize}
    \item Es sollen Schülerdaten (Vor-, Nachname, Id, Klasse und Klassenlehrer),
    sowie Informationen über Besuche im Trainingsraum (Datum, Uhrzeit, Lehrer,
    Anzahl der Besuche) abgerufen und in einer grafischen Oberfläche auf dem
    Bildschirm ausgegeben werden.
	\item Alle Daten (inkl. Lehrer, Klassen) sollen gefiltert und sortiert werden
	können.
	\item Nur zugänglich für den Admin
  \end{itemize}
  \item Zeitzähler (Priorität 2)
  \begin{itemize}
    \item Es soll ein Zeitzähler realisiert werden, der die Aufenthaltszeit
    jedes Schülers, beginnend bei 0 Minuten hochzählt und vor Erreichen von 15
    Minuten mit einer roten Markierung hinterlegt ist. Bei Erreichen von 15
    Minuten soll die Hinterlegung von rot auf grün umspringen und ein Button
    realisiert werden, mit dem die betreuende Lehrkraft den Schüler (abhängig
    von seinem Rückkehrplan) aus dem Trainingsraum entlassen und aus der Liste
    entfernen kann.
	\item Der Zähler soll solange weiterlaufen, bis der Button gedrückt wurde und
	der Schüler den Raum verlässt.
	\item Kehrt der Schüler aufgrund einer Rückkehrplan-Ablehnung zurück, beginnen
	neue 15 Minuten, was über einen Rückkehrer-Button realisiert werden soll.
	\item Wenn das Programm abstürzt, sollen vorhandene Daten von aktuellen
	Trainingsraumbesuchern nicht verloren gehen. Wird das Programm nach einem
	Absturz wieder gestartet, sollen diese Daten aktualisiert vorhanden sein.
  \end{itemize}
  \item Druckfertige Nachricht (Priorität 2)
  \begin{itemize}
    \item Ist ein Elterngespräch oder eine Konferenz notwendig, soll eine
    druckfertige Nachricht mit unveränderbarem Template erstellt werden können.
	\item Das Template für diese Nachricht soll abhängig von der Schule gestaltet
	werden können
  \end{itemize}
  \item Neuanlage Lehrer mit hinterlegter Mailadresse (Priorität 3)
  \begin{itemize}
    \item Neue Lehrer sollen nur vom Administrator angelegt werden können, wobei
    hier ebenfalls eine Emailadresse hinterlegt werden soll.
  \end{itemize}
  \item Mailfunktion (Priorität 5)
  \begin{itemize}
    \item Es soll eine Emailfunktion vorbereitet werden, die eine Info an die
    beteiligten Lehrer versendet, sobald ein Schüler den TR zum dritten, bzw.
    fünften Mal betritt.
  \end{itemize}
\end{itemize}
User Stories Unterstützungsfunktionen
\begin{itemize}
  \item Statistiken anhand der Exportdaten (Priorität 3)
    \begin{itemize}
    \item Anhand der exportierten Daten sollen lokal Statistiken erstellt werden
    können (Filter- und Sortieroptionen). 
  \end{itemize} 
  \item Externer Zugriff (Priorität 5)
  \begin{itemize}
    \item Für die Administration soll ein externer Zugriff auf die Daten über
    eine zusätzliche Software Komponente möglich sein.
	\item Diese Software ist durch zusätzliche Sicherheitsmechanismen zu schützen.
  \end{itemize} 
\end{itemize}
